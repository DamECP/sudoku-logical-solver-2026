Technique claiming

4. Nettoyer et simplifier Unit

Éviter les méthodes redondantes (in_rows, in_cols, etc.).
Garder des méthodes métier centrées sur les besoins des techniques (cells_with_candidate, boxes_with_candidate, etc.).
Renommer éventuellement count_candidates() car elle ne compte pas vraiment.

Nettoyer les annotations et imports

Supprimer les imports inutilisés (Cell, Unit, Iterable dans certains fichiers).
Corriger les annotations (dict[int, Unit] plutôt que dict[int, list[Cell]]).
Garder les types cohérents partout.

Faire remonter plus de logique dans les classes

Les techniques doivent exprimer les règles Sudoku.
Unit et Cell doivent gérer les opérations bas niveau.

Exemple visé :

cell.solve(value)
unit.cells_with_candidate(candidate)
unit.boxes_with_candidate(candidate)

plutôt que :

[c for c in unit.cells if candidate in c.candidates]
